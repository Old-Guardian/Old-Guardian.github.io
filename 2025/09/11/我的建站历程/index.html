我的建站历程

发表于2025-09-11|更新于2025-09-11|技术
|总字数:3.2k|阅读时长:11分钟|浏览量:|评论数:

这是我的第一篇博客，在此记录本站的建站历程。虽然称不上什么正经教程，但也希望能给后来想要建站的人们提供一点点参考，以及我踩过的坑😇

所有参考过的教程都在文中列出了网址，以供参考。

Hexo+Next主题

我的第一个博客站点使用的是Next主题，不得不说网上关于Next主题的教程虽然很多，但相当一部分都稍显老旧，很多时候不得不依靠多篇教程交叉参考。相比之下Butterfly主题的教程就显得完善许多。

Next基本设置

在准备开始安装各种框架程序阶段，我主要参考了如下两篇教程：

  • GitHub Pages + Hexo搭建个人博客网站，史上最全教程
  • 2024年，如何使用 github pages + Hexo + Next 搭建个人博客

第一篇教程非常详细，几乎是从零开始手把手教你怎么开始搭建博客，但缺点是发布于2021年，距离现在比较久远，个人只将它当作一个大致步骤的参考。

相比之下第二篇教程同样很细致，而且距离现在很近，唯一的缺点就是没有前者那么详细，我个人搭建的时候就是按照这个教程来的。但是在用这个教程的时候千万要注意，该教程中的 “Step3：配置 Next 主题” 里的命令 git clone https://github.com/iissnan/hexo-theme-next themes/next 是无效的，主要原因在于 Next 主题似乎迁移了一次官方 Github 网址，导致上述网址访问是 404 ，这里建议去现在的官方网址，里面包含了中英的安装指导。

接下来就从头到尾地分享我个人的建站过程。

首先是 Github 账号和 Git 的安装，这一步网上有大量的资料，在此不赘述。

然后是安装 NodeJS ，这里我参考了NodeJS安装及配置(Windows)，它的 “一、下载安装包” 部分是没什么问题的，但 “二、修改全局包路径” 对于个人而言并不管用，而且会报错使得 Hexo 无法安装，且卸载重新安装 NodeJS 并不会改变被修改过的环境变量，在执行npm root -g命令时依旧会显示被手动修改后的路径。

安装好后，我就顺着前文提到的教程完成了 Github 仓库的创建和 Github Pages 的示例使用。

Next站点美化

到这里站点和主题就算是搭建好了，接下来就是进行站点美化，我参考的教程有：

  • Hexo的Next主题优化教程
  • Hexo Next 主题安装、配置及美化

这两篇教程都非常详细，涵盖了美化的许多方面。因为是第一次建站，所以我并没有一下子引入里面的全部功能，所以只分享一下我配置部分功能的过程。

在配置之前，首先要分清站点配置文件和主题配置文件，二者的文件名都一样，但所处目录位置不同，在反复修改这俩文件的时候注意别弄混了：

  • 站点配置文件：位于你的 Hexo 根目录下，文件名_config.yml；
  • 主题配置文件：位于你的Hexo根目录\themes\next下，文件名同样是_config.yml。

其他的配置教程里都讲得非常详细了，我下面只分享一下踩过的坑。

在Hexo的Next主题优化教程中，里面的 “添加站点访问人数计数” 部分，用的是不蒜子计数。当你按照流程部署完毕，在本地预览时，你也许会发现页脚的浏览人数非常大。这个是正常现象，参考hexo页脚添加访客人数和总访问量中所说，只要将博客部署至云端就可以恢复正常。

此外，在自己查阅其他关于部署不蒜子的教程时，一定要注意里面不蒜子的地址。因为不蒜子曾经更换过一次域名，所以一些古早教程里可能还使用的是旧域名，具体参考不蒜子官方网址。

实际上，无论是在站点配置文件或是主题配置文件中，注释条目里都列出了相对应的官方网址，建议在添加或修改某个功能模块时先查阅一下官方网址，避免被过期教程误导。

除了不蒜子外，在Hexo Next 主题安装、配置及美化中还详细介绍了 LeanCloud 的计数方法，这里也分享一些我遇见的问题和解决过程。

首先是在 “文章阅读量、访问量计数” 部分，按照原教程的配置后，我遇到了文章阅读量在本地预览时不显示，云端部署时显示 “Counter not initialized! See more at console err msg.” 的问题。针对这一问题，我查询到了两个可能的解决方法：

  1. HEXO博客使用LeanCloud报错“Counter not initialized! See more at console err msg.”：这篇文档里的解决方法亲测对我来说不管用。
  2. 解决使用LeanCloud国际版时网站文章浏览次数不显示的bug：这篇文档里的方法对我来说是可用的，修复了上述文章阅读量不显示的问题。

然后是在 “页脚添加总访问量” 部分，按照教程的指导配置后，我遇到的问题是无论在本地预览还是部署到云端，页脚的总访问量一直显示 “本站总访问量 Loading…” 。由于本人不熟悉 Java 的编写，所以以下解决方案来源于 AI ，对我来说是管用的，仅供参考：

Hexo+Butterfly主题

如前文所述，Butterfly主题的教程大多比较新，而且官方教程写得十分完善。基本上依靠官方教程就能搭建出一个很漂亮的站点了，不过极个别美化还是需要自己找其他教程参考，下面我分享了我参考过的教程以及过程。

Butterfly基本设置

在基本设置上我主要参考了如下两篇教程：

  1. Butterfly 文档(一) 快速开始：这是Butterfly官方教程的第一篇，在页面的右下角你可以找到系列文章，只要能细细读完前三篇，基本上就能搭建起一个漂亮的站点了。
  2. Hexo的Butterfly主题配置美化与部署：相比事无巨细的官方文档，这篇教程要更加便捷清晰，如果觉得官方文档有点无从下手的话可以参考这篇。

Butterfly美化

绝大部分Butterfly的美化设置都可以在官方文档中的第三篇Butterfly 文档(三) 主题配置 中找到，这里我仅仅记录两个个我用到的其他美化教程：

  • Butterfly主题一图流背景及文章顶部图修改：这个教程打造了类似我个人主页的效果，即将页面头图和站点背景合二为一，实现更加整体通透的效果。
  • Hexo+Butterfly主题设置背景透明度：这个教程实现的是博客页面的透明度调整，只不过个人在实践过程中发现其中调节文章页背景、所有页面背景和侧边卡片透明度的设置好像有点问题，具体表现是只要更改了侧边卡片的透明度，所有页面的透明度都会被更改，单独设置文章页背景透明度好像也不起作用。暂时没有找到解决方案。

此外，关于文章加密部分我参考了为Hexo文章设置加密这篇教程，其中遇到的一个小问题是在手机端无法输入密码，具体表现在只要按手机键盘上的回车，就会自动跳到评论部分的第一行。我的解决办法是对于加密的文章，将 comment 选项设置为 false ，这样就能正常输入密码查看了。

总之，以上就是我两次建站的经历，不能算教程，只是经验分享，也不保证适用于每个人。祝愿大家都有愉快的建站旅程！
